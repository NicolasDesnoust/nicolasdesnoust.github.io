Wordsearch

  • langages
    Java, Gherkin
  • frameworks
    Spring Boot
  • librairies
    Cucumber, ArchUnit, Tesseract
  • concepts
    Clean architecture, Reconnaissance optique de caractères, Multi-stage Docker builds

published: true

Wordsearch est une application web qui permet de résoudre des grilles de mots-mêlés. Cette application a principalement été conçue pour expérimenter les technologies open source capables d’effectuer de la reconnaissance optique de caractères et d’évaluer leur viabilité en production. L’application est développée selon une variante de la clean architecture, afin de maximiser la réutilisabilité de la logique implémentée.

Ce projet me permet également d'éxpérimenter :

  • La rédaction de tests d'acceptation découplés avec Cucumber.
  • La rédaction d'une documentation facile à maintenir car intégrée au code et générée en partie depuis ce dernier, avec Antora et Redoc.
  • La conteneurisation des étapes d'Intégration Continue pour faciliter leur débogage avec les builds multi-stage de Docker.

🛠️ Le frontend est en cours de développement.
